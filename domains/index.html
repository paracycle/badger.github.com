Register Domain

Register a domain with the authenticated user. Successfully completing this command will subtract credits from the user's account equal to the number of years the domain is registered for.

  • Command

    POST /domains

  • Parameters (required values are in bold)

    Name Description
    name The domain name being registered. If not available, the command will fail.
    registrant_contact_id The id for the registrant contact.
    name_servers The name servers for the domain.
    privacy Protect whois information, or make public.
    auto_renew Automatically deduct credits when the domain expires.
    years Number of years to register domain. Default number of years is 1. Number of credits deducted from account is equal to the number of years specified.
    administrator_contact_id The id for the administrative contact.
    technical_contact_id The id for the technical contact.
    billing_contact_id The id for the billing contact.

  • Sample Request

    POST https://api.badger.com/domains?name=badger1.com&privacy=true&auto_renew=true®istrant_contact_id=1&name_servers=ns1.badger.com

  • Sample Response

    { "name": "badger1.com", "message": "Domain registered", "expires_on": "2012-11-15T02:49:03Z" }

Update Domain

Update domain information with the authenticated user.

  • Command

    PUT /domains/:domain_name

  • Parameters (required values are in bold)

    Name Description
    registrant_contact_id The id for the registrant contact.
    name_servers The name servers for the domain.
    privacy Protect whois information, or make public.
    auto_renew Automatically deduct credits when the domain expires.
    locked Indicate whether the domain is locked
    administrator_contact_id The id for the administrative contact.
    technical_contact_id The id for the technical contact.
    billing_contact_id The id for the billing contact.

  • Sample Request

    PUT https://api.badger.com/domains/badger1.com?privacy=false&administrator_contact_id=3&technical_contact_id=3&billing_contact_id=3

  • Sample Request (with POST)

    POST https://api.badger.com/domains/badger1.com?_method=PUT&privacy=false&administrator_contact_id=3&technical_contact_id=3&billing_contact_id=3

  • Sample Response

    { "message":"Domain updates complete!" }

Renew Domain

Renew the registration for a domain, extending the expiration date. Successfully completing this command will subtract credits from the user's account equal to the number of years the registration is extended by.

  • Command

    POST /domains/:domain_name/renew

  • Parameters (required values are in bold)

    Name Description
    years The number of years to extend the registration by. Note, you cannot register a domain for more than 10 years.

  • Sample Request

    POST https://api.badger.com/domains/badger1.com/renew?years=2

  • Sample Response

    { "years": 2, "new_expires_on": "2019-01-24T02:45:59Z", "old_expires_on": "2017-01-24T02:45:59Z", "message": "Domain successfully renewed" }

Search

Check to see if a domain is available for registration. Requires authentication.

  • Command

    POST /domains/search GET /domains/search

  • Parameters (required values are in bold)

    Name Description
    query The domain name being searched. Note that .com, .net, etc. are automatically appended to the query.
    serial A user specified serial.

  • Sample Request

    POST https://api.badger.com/domains/search?query=badger1

  • Sample Response

    { "domains": [ [ "badger1.com", true ], [ "badger1.net", true ], [ "badger1.org", true ], [ "badger1.info", true ], [ "badger1.me", true ] ] }

Domain Info

Get basic information about the domain. This does not have to be a domain owned by the authenticated user.

  • Command

    GET /domains/:domain_name/info

  • Parameters (required values are in bold)

    Name Description
    auth_code The auth code for the specified domain. The command returns additional info if an auth code is provided.

  • Sample Request

    POST https://api.badger.com/domains/badger.com/info

  • Sample Response

    { "locked": false, "code": 1000, "statuses": [ "ok" ], "pending_transfer": false, "updated_at": "2011-08-02T02:40:19.0000Z", "registrar": { "name": "Awesome Registrar Inc.", "id": 1337 }, "message": "Command completed successfully", "auth_code": null, "created_at": "2003-08-01T00:39:12.0000Z", "expires_on": "2012-08-01T00:39:12.0000Z" }
